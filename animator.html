ANIMATOR
Fall 2017

Please contact me if you are interested in viewing the source code.

This was a half-semester long project developed in Object-Oriented Design in collaboration with Anika Jagow.

Followed the Model, View, Controller (MVC) design pattern. Created three different view, textual, SVG, and Java Swing view. Also created an interactive view that was a hybrid of all three. This view allowed for user interaction, such as speed control, scrubbing, shape selection. It could also export to an SVG file or textual description.

We also used the Adapter design pattern to fit our model with another team's views. We had to build new controllers that adapted our models for shapes and animations to fit their views.

For this project, I admit that the UI was not a focus, and it was more focused on the functionality and back end. Java Swing was pretty difficult to use, and not the most intuitive to build a view.

For the models, I chose to represent shapes with a Shape interface, and have the Rectangle and Oval classes implement that interface. We used a similar approach with the animations, ChangeColor, Scale, and Move.

Separate controllers for each view, since they were so specific. We used the visitor pattern to implement the SVG view, and read a lot of documentation for Java Swing. Very much so a learning experience.
